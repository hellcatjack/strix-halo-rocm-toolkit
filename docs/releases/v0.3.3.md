# v0.3.3

`v0.3.3` makes the public Huawei Cloud SWR replicas the default stable image
source. Stable release `0.2.0`, ROCm 7.2.1, Python 3.12, PyTorch 2.9.1, and
both qualified image digests remain unchanged.

## Default

`--registry auto` tries anonymous SWR exact digests first and falls back to
GHCR only for acquisition failures. Digest, label, or embedded-lock failures
block immediately.

## Overrides

Use `--registry swr` to require SWR or `--registry ghcr` to require GHCR.

## Project Build Compatibility

Project image creation no longer resolves the external
`docker.io/docker/dockerfile:1.7` frontend. This removes a hidden Docker Hub
dependency after the SWR images have already been verified. An installation
that previously stopped at `PROJECT_INIT` can rerun the identical command.
The installer atomically migrates a Dockerfile only when its SHA-256 matches
the untouched legacy generated template, then resumes without downloading the
stable SWR images again. A user-modified Dockerfile is never overwritten and
must remove any external frontend dependency explicitly.

## Validation Boundary

Automated tests use fake registries. The US development host does not perform a
cold SWR pull. Cold anonymous pull and GPU runtime acceptance are performed on
the China-based Ubuntu 24.04 / Ryzen AI Max+ 395 test host.
