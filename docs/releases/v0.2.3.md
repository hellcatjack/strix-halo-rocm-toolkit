# v0.2.3

`v0.2.3` 是安装器可观测性与恢复体验版本。它不改变正式 GPU 软件基线，不发布新容器镜像，也不改变 ComfyUI 或模型缓存策略。

## 主要变化

- 安装阶段使用稳定的 `PLAN/SKIP/START/DETAIL/WAIT/PASS/SUMMARY` 逐行事件。
- Docker pull、BuildKit、uv/pip、锁定 wheel 下载和 sudo helper 命令可实时显示进度。
- 15 秒无新输出时显示心跳，交互提示期间暂停心跳。
- 新增互斥的 `--verbose` 和 `--quiet` 安装参数。
- 每次运行创建项目隔离的私有日志，控制目录为 `0700`，文件为 `0600`。
- 失败结果统一显示 `CAUSE/STATE/RESUME/LOG`，并保留最多 40 行已脱敏命令尾部。
- 特权 helper 保持 stdout 为严格单一 JSON 协议，所有实时命令输出只使用 stderr。

## 状态兼容

状态 schema 仍为 2。`v0.2.3` 可以接管已经完成 `BOOTSTRAP` 的 `v0.2.2` container 状态：安装器先验证原始 `BOOTSTRAP` 输入摘要，只更新安装器版本、源码 revision、source root 和该阶段摘要。其他阶段摘要不变，已完成动作不会重放。

Full 模式仍只允许在已经到达 `HOST_VERIFY` 或更后阶段时进行同一 `0.2.x` 系列的兼容接管。任何摘要损坏、模式变化、项目变化、目标用户变化或其他输入漂移都会阻断。删除状态文件不是受支持的修复方式。

## 未改变的正式基线

- Python 3.12
- PyTorch 2.9.1
- Stable release ID `0.2.0`
- `ghcr.io/hellcatjack/strix-halo-rocm-python@sha256:e9991f97f578156c8620fbb587d2d34504eb632f165cc5597deaadaa3e692a12`
- `ghcr.io/hellcatjack/strix-halo-rocm-pytorch@sha256:dc0bb217474cfd4f602423bd3bf4fe8714b03e900cf3c6b4417b99e622ebcf8b`

`profiles/releases/stable.json` 原样复用，没有执行镜像 build、push 或 stable manifest 发布。

## 项目与数据策略

本版本不预装 ComfyUI，不增加默认模型目录，不改变 Hugging Face、输入、输出、模型权重或编译缓存的共享策略。每个项目仍由用户在 `amd-ai-project.toml` 中自行选择挂载。

## 升级

```bash
git fetch --tags origin
git switch --detach v0.2.3
./install.sh
```

继续选择原模式和原项目，或重新执行原来的完整 install 命令。安装器会显示状态来源、可信 `SKIP` 阶段和实际恢复起点。日志默认位于 `~/.local/state/strix-halo-rocm-toolkit/logs/`；`v0.2.3` 不自动轮转或删除日志。
