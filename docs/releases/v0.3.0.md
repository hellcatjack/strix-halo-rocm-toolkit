# v0.3.0

`v0.3.0` 是宿主内核与安装安全版本。它不重建正式容器镜像；ROCm 7.2.1、Python 3.12、PyTorch 2.9.1、stable release ID `0.2.0` 及其 exact digest 保持不变。

## 主要变化

- Ubuntu 24.04 完整模式以 OEM 6.17 系列为宿主基线。
- 内核计划和平台计划使用独立审批与独立 digest。
- 内核应用后只有一个 `KERNEL_REBOOT_PENDING` 检查点。
- 重启后先验证显示管理器、Radeon 8060S、`amdgpu`、KFD/render node 和当前启动日志，再准备 Docker、Buildx 和设备组。
- Docker runtime 检查不再错误依赖 Buildx；只有构建和缺失 Buildx 修复路径要求 Buildx。
- 状态升级到 schema 3，记录恢复内核和内核验证证据。

## GTT/TTM 边界

工具不安装 `amd-debug-tools`，不调用 `amd-ttm`，不创建、替换或删除 `/etc/modprobe.d/ttm.conf`，不设置 `ttm.pages_limit` 或 `amdgpu.gttsize`，也不因 GTT/TTM 请求重启。现有 live limit、内核命令行和日志仅作为只读诊断事实；缺失或不同的值不会阻断安装。

平台阶段只处理宿主诊断工具、Docker、Buildx 和设备组，不产生第二次重启。原 `--memory-gib` 参数已从公共 CLI 和特权协议删除。

## 状态兼容

可重建旧 `BOOTSTRAP` 摘要的 `0.2.x` container 状态可以接管。`0.2.x` full 状态会从 `BOOTSTRAP` 重新评估并保留不可变镜像引用，但不会复用旧宿主审批；操作员必须重新审阅内核与平台计划。

## 升级

```bash
git fetch --tags origin
git switch --detach v0.3.0
./install.sh
```

选择原模式和项目目录，按 `CAUSE`、`ACTION` 与 `RESUME` 继续。不要删除状态文件绕过迁移或输入漂移检查。
