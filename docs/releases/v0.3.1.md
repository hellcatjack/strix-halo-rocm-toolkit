# v0.3.1

`v0.3.1` 修复安装器在 `HOST_VERIFY` 或 `KERNEL_VERIFY` 阻断时丢失具体诊断信息的问题。正式容器镜像、ROCm 7.2.1、Python 3.12、PyTorch 2.9.1、stable release ID `0.2.0` 及 exact digest 均不变。

## 修复内容

- `CAUSE` 现在直接列出报告中的 finding code、摘要和 remediation，不再只显示 `host-verify returned change-required`。
- `change-required`、`reboot-required` 和 `blocked` 验证结果可以安全写入 schema 3 状态；未知状态仍被拒绝。
- 修复后重跑原命令时，新的通过报告会覆盖旧阻断状态；已完成的主机写入阶段不会重放。
- 内核验证阻断同样保存 finding code，并继续显示保留恢复内核的处理方式。

## 升级与恢复

```bash
cd "$HOME/src/strix-halo-rocm-toolkit"
git fetch --tags origin
git switch --detach v0.3.1

./install.sh --mode full \
  --project-dir "$PROJECT" \
  --project-name video-lab \
  --image-source pull
```

`v0.3.0` full 状态从 `KERNEL_VERIFY` 起可原地接管，因此已经停在 `HOST_VERIFY` 的安装可直接升级。不要删除状态文件，也不要切换安装模式；读取新的 `CAUSE`，完成其中的 `action` 后继续执行同一命令。
